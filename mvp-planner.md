## 詳細週計畫

### Week 1 — 需求確認 & 環境建置
> 目標：把所有不確定的事情問清楚，環境建好跑起來

- [ ] 整理 Q&A 問題清單，與業主開會確認
- [ ] 確認資料來源（CSV / API / 其他）
- [ ] 確認基金範圍與基準指數
- [ ] 建立 Git repo，設定好分支策略
- [ ] 建立專案目錄結構（frontend / backend）
- [ ] 安裝 Python 環境（Flask 或 FastAPI、NumPy、Pandas）
- [ ] Hello World：Flask 跑起來，前端能打到後端

**週末檢查點：** Q&A 全部有答案了嗎？本地伺服器能跑嗎？

---

### Week 2 — 資料處理 & 後端基礎
> 目標：把資料讀進來，後端能回傳基本資料

- [ ] 確認資料格式，整理成統一的 CSV 或 DB 格式
- [ ] 寫資料讀取模組（載入基金基本資訊 + 歷史 NAV）
- [ ] 建立基本 API 端點：`GET /funds`（基金列表）
- [ ] 建立基本 API 端點：`GET /funds/<id>`（單一基金資料）
- [ ] 建立基本 API 端點：`GET /funds/<id>/nav`（歷史淨值）
- [ ] 用 Postman 或瀏覽器測試 API 回傳正確

**週末檢查點：** API 回傳的 JSON 資料格式是否正確？

---

### Week 3 — 指標計算
> 目標：5 個核心指標全部實作並驗證正確

- [ ] `returns.py`：計算年化報酬率
- [ ] `risk.py`：計算年化標準差、下行標準差、MDD
- [ ] `ratios.py`：計算 Sharpe Ratio、Sortino Ratio、Calmar Ratio
- [ ] `capm.py`：計算 Beta、Alpha（需準備基準指數資料）
- [ ] 建立 API 端點：`GET /funds/<id>/metrics`
- [ ] 手動驗算至少 1–2 檔基金的指標，確認數字正確

**週末檢查點：** 指標數字跟 MoneyDJ 或其他來源對比，大方向合理嗎？

---

### Week 4 — 前端基礎 UI
> 目標：首頁和基金列表頁的靜態 UI 完成

- [ ] 建立共用 `style.css`（色彩變數、排版、字型）
- [ ] 建立 `Header` 元件（Logo、搜尋列、導覽）
- [ ] 完成 `index.html` 首頁靜態版面
- [ ] 完成 `funds.html` 基金列表頁（表格 + 搜尋輸入框）
- [ ] 用 JS fetch 打 API，將基金列表渲染到頁面上
- [ ] 確認列表頁搜尋功能可用（前端 filter 或打 API）

**週末檢查點：** 打開 index.html，列表資料有從後端拿到並顯示嗎？

---

### Week 5 — 單一基金分析頁
> 目標：點進基金後，能看到指標卡和走勢圖

- [ ] 完成 `fund.html` 頁面版面
- [ ] 實作指標卡元件（數值 + 評級 Bar + Tooltip 說明）
- [ ] 串接 `/metrics` API，將 Sharpe / Sortino / MDD / Beta / Alpha 顯示到指標卡
- [ ] 用 Chart.js 或 ECharts 繪製 NAV 走勢折線圖
- [ ] 加入時間切換按鈕（1Y / 3Y / 5Y）
- [ ] 基本 RWD（桌機版至少排版正確）

**週末檢查點：** 單一基金頁所有資料都能正確顯示嗎？圖表有出來嗎？

---

### Week 6 — 投資組合比較功能
> 目標：可以選多檔基金，看對比表和走勢疊圖

- [ ] 完成 `compare.html` 頁面版面
- [ ] 實作「加入比較」按鈕（存在 localStorage 或 URL 參數）
- [ ] 建立 API 端點：`POST /compare`（接收多個基金 id，回傳各自指標）
- [ ] 繪製指標對比表格（多基金並排）
- [ ] 繪製報酬走勢疊圖（多條折線，起點標準化為 100）

**週末檢查點：** 可以選 3 檔基金，並排看到指標差異嗎？

---

### Week 7 — 雷達圖 & 前後端完整串接
> 目標：雷達圖完成，整個流程跑順

- [ ] 繪製指標雷達圖（各軸：Sharpe / Sortino / Calmar / Alpha / -Beta / -MDD）
- [ ] 確認所有頁面的前後端串接正確（沒有寫死假資料）
- [ ] 錯誤處理：API 掛掉或資料缺失時，頁面有合理的提示
- [ ] 統一 UI 細節（間距、顏色、字型大小）
- [ ] 同學分工：各自測試對方負責的功能

**週末檢查點：** 從頭到尾點一遍，流程通嗎？有沒有 console error？

---

### Week 8 — 測試、修 Bug & Demo 準備
> 目標：穩定可 Demo 的版本

- [ ] 完整功能測試（列表 → 基金頁 → 加入比較 → 比較頁）
- [ ] 修掉發現的 bug
- [ ] 確認本地端部署步驟文件（README：怎麼啟動後端 + 開前端）
- [ ] 準備 Demo 腳本（走哪幾個流程、講哪幾個亮點）
- [ ] 錄製備用 Demo 影片（以防現場網路或環境出問題）
- [ ] 確認指標說明文字是否通順（非技術人員也看得懂）

**週末檢查點：** 可以當場 Demo 不卡關嗎？README 是否能讓別人在自己電腦跑起來？

## Buffer & 風險

| 風險 | 對策 |
|------|------|
| 資料來源不穩 | Week 2 先用假資料（mock data）繼續開發 |
| 指標計算數字對不起來 | Week 3 留一天專門驗算，找同學交叉確認 |
| 某週進度落後 | 優先保 Phase 1（基金頁 + 指標卡），雷達圖可砍 |
| Demo 當天環境問題 | Week 8 錄好備用影片 |
